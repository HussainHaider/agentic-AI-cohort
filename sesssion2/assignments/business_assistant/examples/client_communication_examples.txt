================================================================================
CLIENT COMMUNICATION DRAFTER — USAGE EXAMPLES
================================================================================
Feature: draft_client_communication()  /  ClientCommunicationDrafter.draft()

Scenario: Sarah regularly needs to communicate project updates, proposals,
and follow-ups with her clients in a polished, professional way.
================================================================================


────────────────────────────────────────────────────────────────────────────────
EXAMPLE 1 — Project Proposal
────────────────────────────────────────────────────────────────────────────────
Input:
  comm_type   : "project proposal"
  client_name : "Acme Corp"
  context     : "Building a custom CRM integration. 3-month timeline. Budget $45,000.
                 Deliverables: API integration, dashboard, training sessions."
  tone        : "professional"

Expected output:
  Dear Acme Corp Team,

  Thank you for the opportunity to present this proposal. NovaTech Solutions is
  pleased to outline our approach for delivering a custom CRM integration
  tailored to Acme Corp's operational needs.

  PROJECT OVERVIEW
  We propose a three-month engagement encompassing API integration between your
  existing systems and the new CRM platform, a real-time executive dashboard,
  and two structured training sessions for your team. Our timeline ensures
  delivery within your Q3 2026 deadline.

  INVESTMENT
  The total project investment is $45,000, structured across three milestones:
    • Milestone 1 (Month 1): API integration & testing — $18,000
    • Milestone 2 (Month 2): Dashboard development — $18,000
    • Milestone 3 (Month 3): Training & go-live support — $9,000

  NEXT STEPS
  We would welcome a 30-minute call to walk through this proposal in detail
  and answer any questions. Please reply to schedule a time that works for you.

  Best regards,
  [Your Name]
  NovaTech Solutions

EXAMPLE 3 — Response to Client Inquiry
────────────────────────────────────────────────────────────────────────────────
Input:
  comm_type   : "response to inquiry"
  client_name : "BlueSky Logistics"
  context     : "Client asked about data security practices, GDPR compliance,
                 and data storage location for the new platform."
  tone        : "formal"

Expected output:
  Dear BlueSky Logistics,

  Thank you for your enquiry regarding NovaTech Solutions' data security and
  compliance practices. We appreciate your diligence in evaluating these
  important considerations.

  DATA SECURITY
  Our platform employs AES-256 encryption for data at rest and TLS 1.3 for
  data in transit. Access is governed by role-based controls with multi-factor
  authentication enforced for all administrative accounts.

  GDPR COMPLIANCE
  NovaTech Solutions is fully GDPR-compliant. We act as a Data Processor under
  Article 28, and a signed Data Processing Agreement (DPA) will be provided
  as part of the contract. Data Subject Access Requests are fulfilled within
  72 hours.

  DATA STORAGE
  All client data is stored in EU-based data centres (Frankfurt, Germany),
  with no data transferred outside the EEA without explicit consent.

  We are happy to provide our full Security Whitepaper and complete an
  Information Security questionnaire if required. Please let us know how
  we can assist further.

  Yours sincerely,
  [Your Name]
  NovaTech Solutions
